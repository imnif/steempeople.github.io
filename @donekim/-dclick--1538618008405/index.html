[스팀잇 이야기] DCLICK 글쓰기 관련 몇 가지 건의사항

너무나도 예쁜 대문을 선물해주신 @designkoi님께 감사드립니다.

안녕하세요, 디온(@donekim)입니다. 스팀 블록체인 기반 애드센스인 DCLICK의 등장과 함께 하드포크 이후에 다소 움츠려들었던 스팀잇에도 약간의 활기가 도는 것 같습니다 :)

서비스 오픈 후 이틀만에 약 11만 스파가 @dclick에 임대된 것도 그렇고, 100개가 넘어가는 댓글들이 달린 것만 보더라도 스티미언들에게 Proof of Click을 표방하며 등장한 DCLICK은 정말 반가운 존재일 수 밖에 없는 것 같습니다.

아마도 DCLICK에서 사용자 편의 향상을 위해서 지속적인 업데이트 작업 중일 것으로 예상되지만, MONETIZE메뉴를 통해 글을 작성해보고 몇 가지 건의를 해보고자 합니다.

(1) 미리보기 화면 : 먼저 현재 DCLICK에서 포스팅 작성 시에 “>”과 같은 마크다운이나 “center”같은 HTML을 사용하는 경우에 미리보기 화면이 깨져서 나타납니다. (물론 미리보기 화면과 관계 없이 포스팅을 작성하고 나면 포스팅은 제대로 출력되기는 합니다) 미리보기 화면에서도 정상적으로 출력될 수 있으면 좋을 것 같습니당!

(2) 이미지 업로드 방법 : 스팀잇이나 busy에서 포스팅 작성 시에 일반적으로 이미지를 드래그하여 첨부하는 방법을 주로 사용하는데, 현재 DCLICK에너는 이미지를 추가하기 위해서는 하단에 보이는 이미지를 추가 하시려면 클릭하세요 버튼을 통해 파일을 업로드해야 합니다. 드래그를 통한 업로드도 가능할 수 있도록 수정이 되면 좋을 것 같습니다 :)

(3) 지우기 메뉴 클릭 시 팝업창 : 포스팅 하단에 보이는 Clear버튼 클릭 시에 스팀잇과 동일하게 진짜로 해당 내용을 지울 것인지에 대해서 추가로 확인하는 팝업 창을 지원하면 좋을 것 같습니다. (컴퓨터가 중간에 잠깐 버벅거리면서 클릭이 잘못 눌러지는 바람에 열심히 썼던 포스팅이 그냥 날라가버렸네요 ㅠㅠ)

(4) 저자 보상 설정 : 현재 DCLICK에서 포스팅 작성 시에는 저자 보상비율을 선택할 수 있는 메뉴가 없어서 일단 작성을 해보니 기본적으로 50% SBD + 50% SP로 설정이 되어 있는 것 같습니다. 추후 업데이트 예정이시겠지만, DCLICK에서도 저자 보상을 설정할 수 있으면 좋을 것 같네요 :)

(5) 태그 설정 : DCLICK에서 포스팅 작성 시에는 기본적으로 첫 번째 태그로 dclick이 자동으로 들어가기 때문에 최대 4개까지만 태그를 입력해야 합니다. 그런데 위와 같이 그걸 모르고 태그 5개를 풀로 채워서 입력하는 경우에는 6개의 태그가 입력되는 상황이 발생합니다. 포스팅 작성 시 기본적으로 dclick이라는 태그가 추가되며 최대 4개까지의 태그만 입력하라는 안내문구가 삽입되면 좋을 것 같습니다 :)
