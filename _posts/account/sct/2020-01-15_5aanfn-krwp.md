1월 13일 중단되었던 KRWP 베네피셔리 서비스가 금일 UTC 6시부터 재가동 되었습니다.
그동안의 작업 내용은 다음과 같습니다.

1 )  SCT 관련 블록 정보 저장 봇 로직 변경
- SCT 관련 custom json / @sct.krwp 베네피셔리 설정 포스팅 정보와 스팀 보상 정보를 분리하여 저장
- custom json 정보 중 불필요한 정보 제외(claim, auto cliam 등)
- 마지막 블럭과의 차이에 따라 한번에 처리하는 블럭 개수를 가변적으로 처리하도록 수정
  추후 장애가 발생할 경우 빠르게 복구가 가능함
  예) 1000 건 이상 차이날 경우 10ms 마다 100건 씩 입력, 30건 이하 차이날 경우 3000ms 마다 1건 씩 입력

2 ) KRWP 베네피셔리 봇 로직 변경

- 분리된 저장소로 정보를 읽어서 처리하도록 수정

현재 SCT 보상은 모두 처리가 되었고 스팀 보상은 2일 정도의 데이터가 처리되지 않았습니다.
내일 중이면 모든 보상에 대해서 전송 처리가 완료될 것입니다.
